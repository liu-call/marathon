## 一、项目概述

### 1.1 项目背景

本项目面向马拉松爱好者，提供一个集 **赛事信息管理、个人成绩记录、行程花费追踪、表格批量导入、成绩证书 OCR 识别、年度报告生成** 于一体的微信小程序。用户可通过文字、语音、表格上传或拍照识别录入赛事与成绩信息，由 AI 后台解析结构化数据。产品定位为 **专注赛事管理的轻量工具**，不做重社交，跑团功能在后续版本迭代。

### 1.2 目标用户

- **个人跑者（MVP）**：管理自己的赛事、成绩、花费，生成个人年度报告
- **跑团成员（Phase 2）**：加入跑团，与跑友对比成绩、共享赛事信息
- **跑团管理员（Phase 2）**：管理跑团成员、查看团内成绩汇总

### 1.3 核心价值

| 痛点 | 解决方案 |
|------|---------|
| 赛事信息散落在各处，容易遗漏报名/抽签时间 | 统一录入 + 倒计时提醒 + 自动抓取公开赛事 |
| 手动输入繁琐 | 语音输入 + AI 解析 + 表格批量导入 |
| 已有 Excel/CSV 赛事清单需逐一录入 | 表格上传 + AI 智能识别列映射 + 批量导入 |
| 成绩手动录入易出错 | 拍照上传成绩证书，OCR + AI 自动识别成绩 |
| 成绩记录无系统化 | 历史成绩 + PB（个人最好成绩）+ 汇总 |
| 参赛花费不清晰 | 按赛事分类汇总花费 + 年度花费统计 |
| 缺少年度回顾 | 年度马拉松报告自动生成（参赛数、PB、花费等） |
| 跑友之间缺乏互动 | 跑团功能：成绩对比、赛事讨论、团内排行 |

### 2.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 微信小程序原生 | 原生开发，分包加载，适配性最好 |
| 后端框架 | NestJS（Node.js + TypeScript） | 模块化架构，内置依赖注入、守卫、管道，适合中大型项目 |
| 数据库 | MongoDB（Mongoose ODM） | 文档型数据库，与 JSON 数据结构天然契合 |
| 对象存储 | 腾讯云 COS / 阿里云 OSS | 存储证书图片、语音文件、动态图片，永久保留 |
| AI 解析 | 智谱 GLM-4 + GLM-4V（详见 2.4 大模型选型） | 语音解析/结构化解析/OCR/年度感言 |
| 语音识别 | 微信同声传译插件（前端转文字）+ GLM-4（结构化解析） | 分离 ASR 和 NLP，降低成本 |
| OCR 识别 | GLM-4V 多模态大模型 | 成绩证书图片 → 结构化成绩数据 |
| 资讯抓取 | Python + Playwright/requests（独立服务） | 定时抓取公开赛事信息 |
| 数据可视化 | ec-canvas（ECharts 小程序版） | 成绩趋势图、花费统计图、年度报告图表 |
| 定时任务 | node-cron（NestJS 内置 Schedule 模块） | 提醒推送 + 年度报告生成 |
| 用户认证 | JWT（jsonwebtoken）+ 微信 openid 登录 | 无状态认证，水平扩展友好 |
| 权限控制 | RBAC（角色-资源-操作） | NestJS Guard 实现数据鉴权 |
| 部署 | Docker + 腾讯云轻量应用服务器 | 容器化部署，方便迁移和扩展 |

### 2.4 大模型选型对比与推荐

| 维度 | 智谱 GLM-4/GLM-4V | 通义千问 Qwen | OpenAI GPT-4o |
|------|-------------------|--------------|---------------|
| **文本解析** | 优秀 | 优秀 | 最佳 |
| **多模态 OCR** | 优秀（GLM-4V） | 良好（Qwen-VL） | 最佳 |
| **国内访问** | 稳定，无需翻墙 | 稳定，阿里云生态 | 需海外网络 |
| **API 价格** | 低（约 GPT-4 的 1/10） | 中低 | 高 |
| **免费额度** | 有 | 有 | 无 |
| **响应速度** | 快（国内节点） | 快 | 慢（海外） |
| **合规性** | 国内备案，合规 | 国内备案，合规 | 数据出境合规风险 |

**推荐方案：智谱 GLM-4 + GLM-4V**

理由：
1. **性价比最高**：API 价格仅为 GPT-4 的 1/10 左右，适合频繁调用场景
2. **国内访问稳定**：无需海外网络，延迟低
3. **多模态能力强**：GLM-4V 可同时处理文本和图片，一个 API 搞定语音解析、表格映射、OCR 识别
4. **合规无风险**：国内备案，数据不出境
5. **免费额度充足**：初期开发测试阶段免费额度足够

**调用场景与模型分配：**

| 场景 | 模型 | 说明 |
|------|------|------|
| 语音转文字后解析赛事信息 | GLM-4（文本） | 结构化 JSON 提取 |
| 表格列映射识别 | GLM-4（文本） | 表头 + 样例 → 字段映射 |
| 成绩证书 OCR | GLM-4V（多模态） | 图片 → 结构化成绩 |
| 年度报告 AI 感言 | GLM-4（文本） | 全年数据 → 个性化文字 |

#### 3.1.3 状态枚举

#### 3.1.4 语音录入流程

```

    │
    ▼

    │
    ▼

    │
    ▼

    │
    ├── 1. 调用 ASR 将语音转文字
    │
    ├── 2. 调用大模型解析为结构化数据
    │      Prompt 示例：
    │      "请从以下语音转文字内容中提取马拉松赛事信息，

    │
    ▼
```

### 3.3 模块三：赛事资讯展示

#### 3.3.1 功能描述

展示马拉松赛事的报名信息及资讯，来源包括 **自动抓取** 和 **手动录入** 两种渠道。

#### 3.3.2 资讯数据结构

```json
{
  "_id": "资讯ID",
  "title": "2026 北京马拉松报名开启",
  "source": "官方公众号",
  "sourceType": "crawl / manual",
  "content": "正文内容（富文本/Markdown）",
  "raceName": "北京马拉松",
  "registrationUrl": "报名链接",
  "imageUrl": "封面图URL",
  "publishDate": "2026-07-01",
  "tags": ["全马", "金标赛事", "北京"],
  "isOfficial": true,
  "crawlSource": "中国马拉松官网",

  "status": "published / pending_review",
  "createdAt": "2026-07-01T08:00:00Z"
```

#### 3.3.3 资讯自动抓取设计

```
云主机定时任务（每天 08:00 / 14:00 各执行一次）
    │
    ▼
Python 爬虫访问赛事信息源
    │
    ├── 数据源 1：中国田径协会马拉松官网（赛事日历）
    ├── 数据源 2：各大赛事官方公众号（通过 RSS/网页抓取）
    └── 数据源 3：马拉松报名平台（如最酷、跑吧等）
    │
    ▼
解析页面，提取赛事名称、日期、报名链接等结构化信息
    │
    ▼
去重（对比已有资讯，URL + 标题去重）
    │
    ▼
写入 news 集合，status = pending_review
    │
    ▼
管理员在后台审核 → 发布（status = published）
    │
    ▼
推送到小程序资讯流
```

**爬虫服务技术方案：**

| 项目 | 方案 |
|------|------|
| 运行环境 | 轻量应用服务器（Docker 容器） |
| 爬虫框架 | Python + requests + BeautifulSoup / Playwright（动态页面） |
| 调度 | cron 定时任务，每天 2 次 |
| 去重 | 基于原始 URL + 标题 MD5 去重 |
| 存储 | 抓取结果写入 MongoDB news 集合 |
| 合规 | 详见下方合规方案 |

#### 3.3.5 爬虫合规方案

**合规原则：只抓公开数据，不抓正文内容，标注来源，可被溯源。**

| 合规要求 | 具体措施 |
|----------|---------|
| **仅抓公开赛事日历数据** | 只提取赛事名称、日期、报名链接等结构化元数据，不抓取文章正文全文 |
| **尊重 robots.txt** | 爬虫启动前先检查目标站点 robots.txt，被禁止的路径不抓取 |
| **标注来源** | 每条抓取的资讯标注来源网站名称 + 原始 URL，用户可点击跳转原页面 |
| **控制频率** | 单站点请求间隔 ≥3 秒，避免对目标服务器造成压力 |
| **不绕过登录** | 不抓取需要登录/付费才能查看的内容 |
| **不存储全文** | 仅存储标题 + 摘要（前 200 字）+ 链接，正文引导用户去原站阅读 |
| **可被要求下架** | 提供联系方式，原站可要求删除其内容 |
| **UA 标识** | 爬虫 User-Agent 标注身份和联系方式，便于站长识别 |

**数据源合规分级：**

| 级别 | 数据源 | 抓取策略 |
|------|--------|---------|
| ✅ 绿色 | 中国田协官网赛事日历、赛事官网公开页面 | 抓取赛事元数据（名称/日期/链接） |
| ⚠️ 黄色 | 马拉松报名平台（最酷/跑吧等） | 仅抓取赛事列表，不抓取详情页内容 |
| ❌ 红色 | 微信公众号文章 | 不直接抓取，如需接入走官方 API 或人工转载授权 |

**合规声明（小程序内展示）：**

> 本小程序资讯信息来源于互联网公开渠道，仅提供赛事信息索引和链接跳转服务，不存储原文内容。如有版权方认为内容侵权，请联系我们删除。

```
    │
    ▼

    │
    ▼
```

```typescript
// 数据隔离守卫示例：确保用户只能操作自己的数据
@Injectable()
export class DataOwnerGuard implements CanActivate {
  async canActivate(context: ExecutionContext): Promise<boolean> {
    const request = context.switchToHttp().getRequest()
    const userId = request.user.userId  // JWT 中提取
    const resourceId = request.params.id

    // 查询资源，验证 ownerId 是否为当前用户
    const resource = await this.raceService.findById(resourceId)
    if (!resource || resource.userId !== userId) {
      throw new ForbiddenException('无权操作此资源')
    }
    return true
  }
}
```

### 5.4 表格解析服务设计

```typescript
// race-import.service.ts - 表格导入核心逻辑
import { Injectable } from '@nestjs/common'
import * as XLSX from 'xlsx'
import { AiService } from '../ai/ai.service'

@Injectable()
export class RaceImportService {
  constructor(private readonly aiService: AiService) {}

  // 解析表格 + AI 列映射
  async parseSpreadsheet(fileBuffer: Buffer, importType: string = 'race') {
    // 1. 解析表格
    const workbook = XLSX.read(fileBuffer, { type: 'buffer' })
    const sheet = workbook.Sheets[workbook.SheetNames[0]]
    const rows = XLSX.utils.sheet_to_json(sheet, { header: 1 }) as any[][]

    if (rows.length < 2) {
      throw new BadRequestException('表格数据不足，至少需要表头 + 1 行数据')
    }

    // 2. 提取表头和样例
    const headers = rows[0]
    const sampleRows = rows.slice(1, 4).map(row =>
      headers.map((_, i) => row[i] || '')
    )

    // 3. AI 列映射识别
    const columnMapping = await this.aiService.parseText(
      this.buildMappingPrompt(headers, sampleRows, importType)
    )

    // 4. 转换数据
    const mappedData = rows.slice(1).map(row => {
      const obj = {}
      Object.entries(columnMapping).forEach(([colIndex, fieldName]) => {
        if (fieldName && fieldName !== 'ignore') {
          obj[fieldName] = row[parseInt(colIndex)] || ''
        }
      })
      return obj
    }).filter(item => item.raceName)

    return { totalRows: rows.length - 1, headers, sampleRows, columnMapping, mappedData }
  }

  // 批量导入
  async batchImport(userId: string, dataList: any[], importType: string) {
    const model = importType === 'race' ? this.raceModel : this.resultModel
    let successCount = 0, failCount = 0
    const failedRows = []

    for (let i = 0; i < dataList.length; i++) {
      try {
        await model.create({ ...dataList[i], userId, createdAt: new Date() })
        successCount++
      } catch (err) {
        failCount++
        failedRows.push({ rowIndex: i + 2, reason: err.message })
      }
    }

    // 记录导入历史
    await this.importRecordModel.create({
      userId, importType, totalRows: dataList.length,
      successCount, failCount, failedRows, status: 'completed'
    })

    return { successCount, failCount, failedRows }
  }

  private buildMappingPrompt(headers, sampleRows, importType): string {
    const fieldDefs = {
      race: 'raceName, raceDate, raceType, raceLocation, raceDistance, registrationOpenDate, registrationCloseDate, lotteryDate, lotteryResultDate, bibNumber, notes',
      result: 'raceName, raceDate, raceDistance, raceType, finishTime, gunTime, netTime, overallRanking, pace, weather, notes',
    }
    return `用户上传了一个${importType === 'race' ? '赛事' : '成绩'}表格。
      表头：${JSON.stringify(headers)}
      样例数据：${JSON.stringify(sampleRows)}
      请将每列表头映射到标准字段（不匹配映射为 "ignore"）：
      可选字段：${fieldDefs[importType]}
      返回 JSON：{ "列索引": "标准字段名或ignore" }`
  }
}
```

### 9.2 技术风险与对策

| 风险 | 对策 |
|------|------|
| 语音识别准确率不足 | 允许用户手动修正解析结果 |
| 大模型解析不稳定 | Prompt 优化 + 结果校验 + 降级为手动输入 |
| 表格格式多样/列名不标准 | AI 智能列映射 + 用户手动调整映射 + 提供标准模板下载 |
| OCR 识别准确率受图片质量影响 | 引导用户在光线充足环境拍摄 + 支持手动修正所有字段 |
| 爬虫被反爬/页面结构变更 | 多数据源备份 + 定期维护爬虫规则 + 失败告警 |
| 大文件解析超时 | 限制单次导入行数（≤500 行），大文件分批处理 |
| 多用户数据并发写入冲突 | MongoDB 事务 + 前端重试机制 |
| 服务器单点故障 | Docker restart:always + 定期备份 + 预留迁移方案 |
| MongoDB 数据丢失 | 每日自动备份（mongodump）+ 备份文件上传到 COS |
| JWT Token 泄露 | 有效期 7 天 + 支持刷新 + HTTPS 全站加密 |
| 大模型 API 费用失控 | 请求限流 + 每用户每日调用上限 + 监控告警 |
| 订阅消息一次性授权 | 引导用户多次授权，关键节点多次提醒 |

```javascript
// miniprogram/utils/api.js
const BASE_URL = 'https://api.yourdomain.com'

const request = (url, method, data) => {
  const token = wx.getStorageSync('token')
  return new Promise((resolve, reject) => {
    wx.request({
      url: BASE_URL + url,
      method,
      data,
      header: {
        'Authorization': token ? `Bearer ${token}` : '',
        'Content-Type': 'application/json'
      },
      success: (res) => {
        if (res.statusCode === 401) {
          // token 过期，跳转登录
          wx.removeStorageSync('token')
          wx.redirectTo({ url: '/pages/index/index' })
        }
        resolve(res.data)
      },
      fail: reject
    })
  })
}

export default { request, BASE_URL }
```

#### 11.6.2 审核注意事项

|-------------|---------|

| 功能不完整 | 确保所有页面有真实内容 |
| 域名未配置 | 服务器域名已在后台添加 |
